# Client-Server pattern

A kliens kéréseket küld, a szerver válaszokat ad – logika elkülönül a felhasználói felülettől.

## Mikor használd?

- Ha központi adatfeldolgozást és több klienst akarsz.
- Ha webes vagy hálózati rendszert fejlesztesz.

## Példa

- Webbrowser → Webszerver
- API kliens → REST backend

## Előnyök

- Kliens és szerver függetlenül fejleszthető
- Könnyen skálázható

## Hátrányok

- Hálózatfüggőség
- Szerver túlterhelődhet
